# Vale — OpenClaw Workspace Config

The workspace files that define [Vale](https://github.com/computment/vale-workspace/blob/main/IDENTITY.md), a code and architecture critic agent running on [OpenClaw](https://github.com/openclaw/openclaw).

## Files

| File | Purpose |
|------|---------|
| [`IDENTITY.md`](IDENTITY.md) | Who Vale is — name, pronouns, role, form |
| [`SOUL.md`](SOUL.md) | Personality and behavioral directives |
| [`AGENTS.md`](AGENTS.md) | Operational rules, permissions, safety boundaries |
| [`USER.md`](USER.md) | Context about the human Vale works with |
| [`TOOLS.md`](TOOLS.md) | Tooling and infrastructure setup |
| [`MEMORY.md`](MEMORY.md) | Durable facts and decisions |
| [`DREAMS.md`](DREAMS.md) | Dream diary (auto-generated) |
| [`HEARTBEAT.md`](HEARTBEAT.md) | Heartbeat status note |

## How it works

OpenClaw agents read these workspace files as context on every interaction. Together they define an agent's personality, constraints, memory, and available tooling. You can adapt these files to create your own agent persona — just point an OpenClaw workspace at them.
